Kerberos é um protocolo de autenticação de rede que tem como principal função prover maior segurança para aplicações cliente-servidor através de uma chave secreta criptografada.

Há uma série de softwares hoje em dia que possuem suporte ao Kerberos através de bibliotecas e pacotes, como os servidores Apache, Apache 2 e Samba, roteadores e switches Cisco, sistemas operacionais Windows Server e MAC OS X, dentre outros.

Atualmente ele está na versão 5 e suas padronizações podem ser encontradas na RFC 4120, que surgiu como atualização da RFC 1510 da versão 4 do protocolo.

No momento em que é efetuado o acesso em sistema com as credenciais disponibilizadas ou criadas por um administrador, o Kerberos entra em ação e verifica essas informações junto a um servidor, onde comumente é utilizado o Active Directory (AD), que é o diretório responsável por conter os usuários ativos no sistema e que é utilizado em sistemas Windows Server.

O Kerberos trabalha com um sistema de segurança conhecido como "Ticket to Get Ticket", ou TGT. Ele é um arquivo de configuração criptografado que permite acesso a um determinado serviço.

Esse período de aguardo, que normalmente conhecemos como a mensagem de "Bem-vindo" seguido de um loading no Windows, é a etapa onde o Kerberos está verificando suas credenciais junto ao servidor e liberando seu acesso ao domínio.

Caso o usuário pertença ao domínio e a senha esteja correta, o usuário recebe o TGT e tem acesso. Com o TGT disponibilizado, todas as requisições feitas pelo usuário serão feitas diretamente aos serviços do servidor, o chamado TGS: "TICKET GRANTING SERVER".

Agora que sabemos o fluxo seguido pelo Kerberos e temos a noção de usuário comum quanto ao funcionamento do protocolo, podemos seguir para a parte técnica, de processo de instalação e configuração do Kerberos.

Quando configurado corretamente, o Kerberos comunica-se automaticamente com o AD através de habilitação manual no Windows Server, onde essa permissão é liberada na pasta “inetsrv”, que fica localizada no System32 e necessita ser acessada com condições de administrador. Essa pasta é responsável por configurar o sistema de autenticação do Windows.

Um dos motivos por se considerar o Kerberos uma ferramenta completa de se utilizar é quando se opta por migrar um servidor que utiliza outro método de autenticação para utilizar o Kerberos. Essa transferência é conhecida como Kerberizar a aplicação e consiste em algumas mudanças para o seu pleno funcionamento:

- Buscar as informações de dados de usuários já existentes no sistema;

- Cachear essas informações de modo a tornar a requisição mais eficiente;

- Verificar os tickets já solicitados pelos usuários para evitar que tenha de ser repetido o processo para cada serviço já disponibilizado;

A conclusão geral que se pode ter em relação ao Kerberos é que, mesmo sendo um protocolo de autenticação de rede criado há mais de 30 anos, consegue se reinventar em meio às tecnologias atuais e continua sendo utilizado com frequência, devido a sua efetividade.

Sua integração com servidores Windows (Server) e Samba (Linux) é um dos facilitadores que permite que até hoje vejamos ele em diversas redes corporativas e universidades.
